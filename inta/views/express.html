Express课程概述

Express是一个简洁，灵活的node.js Web应用开发框架

第一节：express安装
npm install express 安装express
npm install -g express 全局安装express
使用express：
var express = require('express') //引用express框架
var app = express() //创建express对象
app.listen(3001) //监听端口
第二节：中间件--函数
中间件：express在处理http请求-响应的过程中，执行任务的一系列函数
app.use()是一个中间件函数，此函数接收的参数也是函数，也就是这里说的任务。
app.use(function(){
console.log("hello inta")
})
第三节：req和res
req和res分别代表http的请求对象和响应对象，具体属性和方法请参考下列官方文档：
req官方文档
res官方文档
第四节：中间件--next
在express项目中可以有多个中间件，但需要在中间件的任务函数中加上next参数，并且调用next()才能执行下一个中间件
第五节：路由
路由 = http请求方法 + http请求路径
app.use()中间节接收任何路由
express对象提供了一些方法来接收路由，如：
app.get()
app.post()等
第六节：静态路径
通过express.static(路径)设置静态路劲
设置完毕以后使用app.use(express.static())中间件执行设置
在express框架中，只有两个方法是通过express直接调用的（这里是通过类方法，不是通过对象方法调用的)一个就是本就课的设置静态路径，另一个就是下节课的路由对象
第七节：路由对象
通过express.Router()创建路由对象
路由对象包括get(),post(),put(),delete()等http请求方法
路由对象其实可以说也是一个中间件
路由对象有助于模块化处理http请求-响应
第八节：设置
express通过app.set()设置express应用程序
通过下列网址可以参考express应用程序的设置
http://expressjs.com/en/4x/api.html#app.set
第九节：模板引擎
模板引擎能使页面和数据分离
app.set('view engine',模板引擎) 设置模板引擎
app.set('views',页面路径) 设置页面路劲
app.engine('html',ejs.rendFile) 设置引擎文件
第十节：常用模块
body-parse: 解析http请求的body
    中间件调用:app.use(bodyParser.urlencoded({ extended: false }));
cookie-parse: 解析http请求的cookie
    中间件调用:app.use(cookieParser())
server-favicon: 显示标签图标
    中间件调用:app.use(favicon(路径 'favicon.ico')）
morgan: 打印请求信息
    中间件调用:app.use(logger('dev'))
第十一节：生成器
全局安装express生成器： npm install -g express-generator

那年程序员

express让你的Web开发变的一路顺风

“我不在相信任何人”，这是Doug Wilson说的，3年以来一直维护express框架的程序员。
在这么一个和谐的社会，尤其在月亮那么圆的美国，Doug Wilson为什么会这么说呢。因为express这个框架让它的作者TJ Holowaychuk给卖了！没错！ 一个开源的express框架，注意是开源的..被卖了。卖给了StrongLoop公司。Give credit where credit is due, TJ是有可赞之处的，因为我们耳熟能详的Express，Koa，Jade，ejs，Stylus, Mocha, Superagent，Commander等等，都是由TJ创建的，由他参与的项目更是数不胜数，起码为开源社区奉献了百万行代码，然而正是这个技术大牛的一失足，使他的名声一落千丈。因为我们都知道作为开源express，是由很多程序员共同奉献完成的。（开源代码怎么能卖，大家去看MIT吧）你这么一卖，等于葬掉了别人的情怀，而Doug Wilson就是被坑的人。而粘上铜臭的express在卖给StrongLoop的两年内没有得到一次pull request,最终移交给了Node Foundation。

摘自 --- 《The Unbelievable History of the Express JavaScript Framework》
