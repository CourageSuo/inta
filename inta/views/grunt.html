grunt课程概述

grunt是javascript的项目构件工具，为你javascript项目构件提供一些精妙的解决方法

第一节：应用案例
介绍grunt应用案例：
压缩html文件
压缩javascript文件到一个文件夹下
把sass文件编译为css文件
第二节：简介和安装
使用grunt需要安装：grunt.js npm grunt grunt-cli grunt-plugin
全局安装grunt-cli: npm install -g grunt-cli
在项目下安装grunt: npm install grunt --save-dev
一般情况下载javascript项目中存在package.json文件，我们把代码中用到的包卸载package.json中的dependencies中，把开发中用到的包写在devDependencies中。因为grunt属于开发中使用的包，故存放在package.json的devDependencies中
第三节：gruntfile.js
gruntfile.js用于：（说出他是一个很重要的文件),若使用grunt，就必须创建两个文件：
package.json 和 gruntfile.js
gruntfile.js的作用：
加载插件（plugin）
配置插件
定义或者说注册任务
grunt文件内部是一个包装函数，此函数运行grunt时被调用
第四节：对象
在gruntfile.js的包装函数中，有一个参数，此参数就是grunt对象。
可以通过typeof(grunt)查看参数类型
可以通过console.log(grunt)打印此参数
第五节：config属性
config属性的使用
为当前的项目初始化配置对象
grunt.config.init 初始化配置对象
grunt.config 读取配置对象 等
可以通过grunt接口文档查询--http://gruntjs.com/api/grunt.config
第六节：file属性

grunt.file.read(文件目录) 读取文件
grunt.file.readJSON(文件目录) 读取JSON文件
grunt.file.write(文件目录，内容) 写入文件
grunt.file.copy(源目录，目标目录) 写入文件
grunt.file.delete(文件目录) 删除文件
grunt.file.expand([参数]，匹配文件) 返回文件和目录的数组
可以通过grunt接口文档查询--http://gruntjs.com/api/grunt.file
第七节：task属性
task用于加载，注册，执行任务
grunt.task.registerTask(执行名称, 任务列表) 执行任务
grunt.task.loadTasks(任务路径) 加载任务
grunt.task.loadNpmTasks(插件名称) 加载npm中的任务
第八节：创建插件
一个插件可以有多个任务，一个任务是一个包装函数
使用grunt.task.loadTasks（任务路径）加载创建的任务
使用grunt.registerTask 执行任务
第九节：发表插件
1.安装grunt-init npm install -g grunt-init
2.安装grunt插件模板
    git clone git://github.com/gruntjs/grunt-init-gruntplugin.git ~/.grunt-init/gruntplugin
3.grunt-init gruntplugin 初始化grunt插件 （在一个新的目录中）
4.安装package.json依赖文件 npm install
5.发表grunt插件 npm publish
第十节：压缩html和css
压缩html插件：grunt-contrib-htmlmin
压缩css插件：grunt-contrib-cssmin
使用npm中grunt插件，gruntfile.js文件的基本模板：
1.初始化配置对象
2.加载npm插件
3.注册任务
grunt-contrib-htmlmin参考文档
grunt-contrib-cssmin参考文档
第十一节：压缩javascript
grunt-contrib-uglify插件： 压缩javascript文件，并把多个js文件压缩到同一个js文件下
grunt-contrib-uglify参考文档
第十二节：编译sass
grunt-contrib-sass插件： 把sass文件编译成css
grunt-contrib-sass参考文档
第十三节：watch
watch插件：当文件改变时运行相应任务
grunt-contrib-watch参考文档
