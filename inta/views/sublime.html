Sublime课程概述

一个具有漂亮的界面和强大的功能的代码编译器

第一节：常规操作
在项目文件夹中，单击选择文件，双击把文件添加到标签栏
control + z 撤销操作，control ＋ y 取消撤销的造作
control + alt + m 选取大括号中的内容（Mac:control + shift + m)
control + shift + a 选取两个标签中的内容(Mac:shift + command + a)
control + 放大字体
control - 缩小字体
通过鼠标右键查看文件更改前后区别
第二节：窗口 视图 主题
状态栏右下脚可以改变目标代码语法格式
通过菜单栏：view(视图) -> layout(布局) 增加编译窗口
通过Preference(偏好) -> Color Scheme(色彩组合）改变窗口样式【Mac:在菜单栏sublime text下，Windows在菜单栏Preferences下】
第三节：行操作
control + l 选择当前行的内容
control + alt + ⬆️和⬇️ 上下移动当前行的内容(Mac:command + control + ⬆️和⬇️)
control + [ 和 ] 左右缩进当前行的内容
control + delete 删除当前行的内容
第四节：光标操作
按住control点击鼠标左键可以操作多个光标
按住alt拖动鼠标可以把光标拖动到多行
选中按住control + shift + l 行尾操作多个光标
把行尾多个光标移动到行首(行尾），点击home（end）键（Mac：command + 左箭头或右箭头)
这次课程录时紧张了，两处口误：1，多个光标只能输入相同的文字；2，最后是把光标移动到行首
第五节：缩进
选中缩进区域，通过Eidt(编辑) -> line(行) -> Reindent(递归缩进)
control + shift + v 可以智能缩进粘贴
第六节：查找
control + f 普通查找
control + d 查找光标所在字符
control + i 查找后快速更改
control + alt + g 选中全部查找内容（Mac：control + command + g)
control + shift + f 查找指定文件夹中所包含的内容
control + g 查找行号 (Mac: control + g)
第七节：究极查找
control + p 打开究极查找输入框
输入“文件名” 查找整个sublime中的文件
输入“文件名” + @ 查找文件中的方法或选择器
输入“文件名” + # 查找文件中的内容
输入“文件名” + ： 找到对应的行号
Esc退出究极查找
第八节：控制板
control + shift + p 调出控制板
Set Syntax: 设置目标代码语法格式
Sort Lines: 按字母排列顺序
Convert Case: 改变大小写
以及其他sublime 控制命令
Esc退出控制板
第九节：右键打开
Windows：
在下方下载区域（那年程序员上方）下载sublime.reg文件，双击后在windows中则可使用右键打开
下载sublime.reg文件
Mac:
1.打开Automater -> service（服务）
2.在左上方搜索框中，输入“shell”
3.把搜索到的运行shell脚本标志拖动到右侧的窗口中
4.在拖入后出现的小窗口中，输入：/Applications/Sublime\ Text.app/Contents/SharedSupport/bin/subl -n $@
5.选择“shell：”后的下拉菜单为：/bin/zsh, 后边的“传递输入”：选择作为自变量
6.“服务收到选定”的下拉菜单中选择“文件或文件夹”
7.取个名字保存，‘command + s’.
8.文件夹右键 -> 服务 -> 所取的名字就能使用sublime打开
第十节：快捷键绑定
打开Preferences -> Key Bindings 文件【Mac:在菜单栏sublime text下，Windows在菜单栏Preferences下】
快捷键绑定标准格式：{"keys",[""], "command":""}
查找命令名称：view -> Show Console 然后执行命令
在输入框中输入sublime.log_commands(True)就可以打印菜单栏命令
sublime.log_commands(False)关闭打印的命令
第十一节：鼠标绑定
通过此网站查看默认鼠标绑定http://folk.uib.no/hab001/Info100/Sublime%20Text%202.0.2/Data/Packages/Default/Default%20(Windows).sublime-mousemap
第十二节：宏
开始记录命令：Control + q
停止记录命令：Control + q
运行宏：Tools(工具) -> Playback（运行宏）
记录宏：Tools(工具) -> SaveMacro(保存宏)
为宏绑定快捷键时需要添加参数，语法为：
{"keys",[""], "command":"","arg":{"file":文件路径"}
第十三节：Snippets
通过 tools -> Snippets 查看当前文件可以使用的snippets
注：不同的文件对应不同的snippets
第十四节：创建Snippets
通过 tools -> developer -> new snippets创见自己的snippets,然后保存为.sublime-snippet形式
<tabTrigger>引导字符标签</tabTrigger>
<scope>对应目标文件语法的snippets</scope>
备注：html文件scope为text.html; javascript文件scope为source.js;
$number: number表示第几个tab光标移到处
${number:stirng}: number表示第几个tab光标移到处,string表示默认光标
第十五节：安装插件
control + shift + p 打开控制板
输入install Package安装插件
输入Remove Package删除插件
第十六节：插件推荐
Emmet插件（适合编译html和css）
如：ul>li*4>a
如：(header.intaHeader>h1+nav>ul>li*5>a)+(div.intaBody>div.main+aside.sidebar)+(footer.intaFooter>p)
如：CSS简写display:block => db; margin-right:10=>mr10;等
AdvancedNewFile插件 （方便保存文件）
如：创建新的文件：command + alt + n 直接输入路径就可以创建文件
SublimeLinter插件 （检查JAVA，javascript，PHP等语言语法错误）
安装完SublimeLinter以后，需继续安装Linter指定的语法检查插件，才能检查相关的语法错误
在安装jshint检查javascript语法时需要安装node.js和jshint模块（npm install -g jshint)
安装完毕以后需要重启sublime
第十七节：创建插件导引
sublime 的插件使用python写的
通过control + ` 打开控制台，可以运行view.run_command('example')
同样可以通过绑定快捷键运行
第十八节：简单认识PYTHON
print (): 打印信息
运行python: command + b
import :引入文件
函数语法：def functionname( parameters ): return value
创建类语法：class ClassName:
          def __init__(self): #构造函数
          def functionname (self,parameters): #定义方法
继承语法： class ClassName(parentCalssName):
第十九节：sublime API（一）
sublime和sublime_plugin是模块
sublime位置：/Applications/Sublime Text.app/Contents/MacOS/sublime.py
sublime_plugin位置：/Applications/Sublime Text.app/Contents/MacOS/sublime_plugin.py
模块就是一个用python代码写的文件，其中包括函数，类和变量，也可以包含可运行的代码
下载Sublime Text API文件
第二十节：sublime API（二）
介绍几个简单的接口方法，这些接口方法，都可以在sublime的官网或INTA翻译的接口文档中找到。
得到光标的当前位置：self.view.sel()[0].begin()
查找文件中的内容：self.view.find("匹配的字符", 从哪个位置开始匹配)
把光标移动到指定位置：self.view.sel().add(Sublime.Region(a,b))
第二十一节：添加菜单按钮
在菜单栏中添加按钮：Main.sublime-menu
在边栏中添加按钮：Side Bar.sublime-menu
鼠标右键添加按钮：Context.sublime-menu
在sublime其他位置添加按钮
第二十二节：发布插件
一般情况下，我们会通过git把开发的插件合并到sublime的Package Control Channel中，成功合并后，就能直接在控制板中下载
1. 把sublime包文件上传到github仓库中
2. FORK Package Control Channel(参见git课程)
3. 通过git克隆FORK到本地计算机中
4. 使用sublime 打开package_control_channel文件夹,在repository文件夹中，把创建的plugin信息到对应的文件中（可参见视频）
5. 安装 ChannelRepositoryTools 包并运行Test Default Channel command 成功后进入下一步
6. 创建pull request

那年程序员

说它免费它也不免费，说它轻量级它也不轻量级

2007年，Jon Skinner在google的工作都不干了，就为了追寻他的中国梦，错了，是他的美国梦。就是创建一款简洁，美观，漂亮，大方，易操作，易理解，易学习轻量级的文本编辑器 -- Sublime Text。当然对于一个google的软件工程师来说开发一个小小的文本编辑器还是信手拈来的，关键在于，如何开发一个让程序员使用起来比较舒服的文本编译器。其实sublime text的整体设计围绕的就是两个字“舒爽”。节约出屏幕大部分的空间都是用于文本编译，不像有些编辑器，整个窗口都在卖弄自己的功能，这些功能扎根在屏幕的每一个角落，给人的感觉好不复杂。而Jon Skinner知道给用户怎样的体验，而这种体验也正是INTA课程的思想模式:呈现最本质的内容，一切皆为易。
