Git课程概述

Git是开源的，分布的版本管理系统

第一节：版本管理介绍
版本管理系统简而言之就是记录文件或文件夹的变化，以便一段时间过后找到对应每个变化的文件或文件夹。
第二节：Git基础
如何最基本的使用git进行版本管理
git init 开始管理你的项目和文件夹
git add 文件名 把文件加到暂存台
git commit -m “描述信息”
git checkout 描述符 切换到任何快照
git checkout master 返回最新快照
git tag -a 版本号 -m '版本信息' 提交版本信息
git checkout 版本号 切换相应的版本
每次提交都会产生一个快照。git通过快照存储文件的相关信息。
第三节：git状态
git status 显示当前git的状态
git中文件的两种状态：已跟踪(tracked)和未跟踪(untracked)
git rm --cached 文件 把已跟踪文件转为未跟踪文件 1./etc/gitconfig -- system 2. gitonfig /.config /git/config --global
第四节：暂存台操作
git diff 未跟踪文件和暂存台中文件的不同
git diff --staged 查看暂存台中的文件和最后一次提交的不同
git commit -a -m ‘描述信息’ 对于已在暂存台中的文件，更改后可以通过此命令直接提交。
git rm -f 文件名 从暂存台和当前目录中删除文件
git rm --cached 文件名 只在暂存台中删除

注：视频中讲解删除暂存台中文件时，视频中的话语是‘工作台’，应改为暂存台。

第五节：撤销更改和重复提交
git checkout -- 文件名 撤销更改
git commit --amend -m '描述信息' 覆盖提交
第六节：日志信息
git log -p 展示每次提交的不同
--stat 简单展示git的状态
--pretty=参数 格式化日志信息
--since=时间 展示指定提交时间后的信息
--graph 可以和pretty=oneline一起使用展示分支和合并的信息
第七节：主分支
git分支有着git特有的运行机制，本节主要讲述git主分支(master)与快照的运行机制。
第八节：分支
git branch 分支名 创建分支
git checkout 分支名 切换分支
git log - - decorate 查看当前分支
git checkout -b 分支名 直接创建加切换
第九节：提交分支
git log - - oneline - - decorate - - graph - - all
我们可以通过此命令图表化查看分支信息及分支的提交情况
第十节：合并分支（上）
合并分支分两种方式：
第一种是 不创建快照合并
在master分支最后一个快照，创建的分支。合并后只需把HEAD指针直接移到合并分支的最新提交处。
此方法官方称之为:Fast-Forward
git merge 分支名 合并分支
git checkout -d 分支名 删除分支
第十一节：合并分支（下）
第二种是 创建快照合并
在master一个快照创建分支后，继续对master分支中的文件进行更改提交，合并时，git会创建新的快照合并分支。
此方法官方称之为：Three-way
第十二节：合并冲突
在不同分支中对同一个文件进行修改，会产生合并冲突

git status 查看分支中哪个文件产生的合并冲突

第十三节：多分支及合并
git作为版本管理系统，能够技压其他版本管理系统的主要原因就是它的分支创建和分支合并特性，在编写大型的项目中，此特性尤为突出。
第十四节：分支管理
git branch 显示当前分支
加 -v 显示分支最后提交的一次快照
加 --merged 查看哪个分支已经合并
加 --no-merged 查看哪个分支没有合并
加 -D 强制删除分支
第十五节：配置和帮助
git config 用来设置git需要的参数
git config --global user.name "inta" 设置git用户的名称
git config --global user.email "inta@intalesson.com" 设置git用户的邮箱
git config --list 查看git配置信息
可以通过帮助，查看git的命令
git help 命令
git 命令 --help
man git-命令
第十六节：Github简介
git是版本管理系统
github是服务器，是存放项目版本的服务器。
我们可以通过www.github.com登录github.
项目放到github上的优势:
1). 可以防止项目代码丢失
2). 可与其他开发者协作开发
以及如何在github上创建和删除仓库
第十七节：连接github
git remote add 连接名称 url地址
git remote 查看连接
git remote -v 连接名字对应的url地址
git remote show 连接名称 查看连接的详细信息
git remote rename 就名称 新名称 重新命名连接名称
git remote rm 连接名称 删除连接
第十八节：上传github仓库
git push 连接名称 分支名称 把分支上传到github仓库中
git push 连接名称 tag名称
在github创建完仓库的引导页上，使用的是git push -u进行上传，此知识点会在二十节《跟踪分支》讲解
第十九节：下载github仓库
git clone 仓库url地址
此命令会自动创建名字为origin的连接
git fetch 连接名称 下载的文件成文本地的一个远程分支
git branch -r 查看远程分支 这里望大家参考二十一节知识点，这里的远程分支，就是二十节远程跟踪分支
git checkout -b 分支名字 远程分支 切换远程分支
git pull 连接名称 github分支 下载并和并分支
git pull = git fetch + git merge
第二十节：github分支
github中的【branches】分支按钮可以在github中创建分支
github中的【crate new file】创建文件按钮创建新的文件
github中的【upload file】 上传文件按钮上传文件
github中的【find file】 查找文件按钮查找文件
github中的 【tags】 标签按钮查看文件
注：使用git clone, git fetch, git pull在本地得到是github中的master分支，想要得到在github中创建的分支，需要查看下节（二十一节github课程）《跟踪分支》知识点。
第二十一节：跟踪分支
使用git fetch从github上下载的项目（文件）在本地计算机会以一种远程跟踪分支（remote-tracking branches)的形式存在，名称为（连接名称/github分支),上节视频讲过，可以通过git branch -r查看远程跟踪分支。我们不能在本地修改或合并该远程跟踪分支，可以通过下面命令创建一个新的分支进行修改或合并：
git checkout -b 跟踪分支 远程跟踪分支 把远程跟踪分支转化为跟踪分支
跟踪分支和github的分支有着直接的关联，也就是说在本地跟踪分支中，直接使用git pull，git就会直接找到对应的github服务器，并更新合并分支中的内容。
如果使用git clone下载仓库，下载完毕后本地的master分支，就是跟踪分支。
在使用git push上传本地项目到github时，如果使用'git push -u 连接名称 分支名称'，在github下载后的分支就会变为跟踪分支.
git branch -r 查看远程跟踪分支
git branch -vv 查看跟踪分支
第二十二节：合并申请
git是一个免费开源的分布式版本管理系统
通过github中的fork可以直接复制github中的其他项目变成自己的项目
对github中别人的项目更改完毕以后可以通过pull request（合并申请）把添加的新功能合并到他人的项目中。实现开源软件世界性的共同开发。
