NPM课程概述

npm是javascript的包管理工具,我们可以通过npm查找和复用其他开发者的代码，同样，我们也可以通过npm分享自己的功能模块供其他开发者使用，共同提升开发效率。

第一节：npm简述
npm最初是node package manager的缩写，也就是node.js的包管理工具。现在npm已经发展为javascript的包管理工具。
npm和node绑定在一起安装的，可以通过npm install npm@latest -g更新npm的最新版本
第二节：npm基本使用
本节讲述如何安装包文件，及其相应的版本。如何查找包文件的安装路径。安装包文件后，会在当前目录下创建node_modular路径（文件夹）
npm install 包名称
npm ls 查看当前项目中安转的包文件
npm uninstall 卸载包
npm install 包名称 -g 全局安装包文件
npm ls -g 查看全局的包文件
npm root (-g) 查看包的安转路径（或全局安装路径)
第三节：npm递归安装
在安装比较复杂的包文件时，可能会依赖其他的包文件。npm通过package.json安装依赖的包文件。我们可以通过：
npm ls --depth=<数字>
简化的查看安装包文件的属性结构
第四节：package.json
npm通过package.json简便快捷的安装包文件，及其依赖的包文件。顾名思义，package.json文件内容是一个json串。package.json必须包含"name"和"version"键。
name: 包的名称
version: 包的版本
main: 指定包的入口文件，默认是index.js
script: 指定npm脚本命令
keyword: 便于搜索引擎搜索的关键字
author: 作者
licence: 认证许可
repository: 包位于git的连接地址
第五节：自动创建package.json文件
简化创建package.json的方法
npm init
npm init --yes 自动创建package.json并使用其默认的值
第六节：dependencies
在package.json的dependencies键中添加相应的依赖包。就可以通过
npm install
直接安装其依赖的包文件
"*" 和 "" 表示最新版本
"@x.x.x" 表示对应的版本
第七节：devDependencies
devDependencies是表示开发中依赖的包文件
个人理解：其实就是开发过程中，用到的一些用javascript写的一些工具，如编译sass，缩小js文件等
第八节：package.json添加包
npm install 包名称 --save 向package.json中dependencies中添加包
npm install 包名称 --save-dev 向package.json中devDependencies中添加包
第九节：发表包
创建完package.json文件后，可以通过
npm publish
发表自己创建的包分享给其他开发者使用。在这里需要注意npm会将当前目录中的所用文件发表的npm中，发表之前需要添加用户。 如果想删除包使用
发表之前使用npm adduser 添加用户
npm unpubish --force 包名称
对其进行删除（限24小时内操作）
注：本视频index.js代码是funciton 应为function
第十节：包和模块
node.js和npm中，包和模块的定义很容易混淆。
包是一个被package.json描述的一个文件夹或路径。
模块可以被node.js中的require引用。
大多数的包都是模块
包中的package.json有main键
包中包含index.js
单独的javascript文件

那年程序员

javascript, javascript, javascript.

npm的作者是Isaac Z. Schlueter（艾萨客·施吕特），看这个名字应该是个犹太人，它利用了犹太人独有的天赋与智慧一两天就把npm给完成了。写完以后继续对npm进行开发和维护。当初，他也写了一份js包文件在npm上的发布标准，但是对于初创阶段的艾萨客，根本就没人鸟他。它那时也没有创业资金，资源也有限。但是像很多成功的故事一样，他还是坚持不懈，自己开发javascript的包放到npm上，免费供别人使用。然后，npm从最初的十几个包，到现在的成千上万个包。就连艾萨客自己也没想到npm有如此迅猛的发展趋势，以后的发展差不多就都一样了，就是开办npm公司，出任CEO，跻身高富帅，赢取白富美，走向人生巅峰了...
