Backbone课程概述

一个轻量级的前端类库

第一节：简介
backbonejs是一个轻量级的类库
类库：类的集合
框架：类，数组，变量等等相关类型的集合
Backbone.js中主要的类有Model View Collection Event Router History Sync
使用backbone需要引入undersocre.js 和 JQuery
jquery.min文件下载
underscore-min.js文件下载
backbone-min.js文件下载
第二节：类和对象
在Backbone中使用 new 关键字创建对象
第三节：继承
继承就是子类可以使用父类的属性和方法
在Backbone中，使用Extend实现继承
【可以说A类继承自B类，亦可以说A类扩展自B类】
第四节：View
属性：uid每一个View对象都有唯一的uid
属性：el表示当前元素；默认是div
属性：$el表示可以看成是‘$’和‘el’，也就是使用JQuery语法表示当前元素
initialize: 创建一个对象时会默认调用此方法
tagName: 设置当前元素
第五节：模板
模板：就是用同样的代码显示不同的内容
使用<script type="text/template"></script>创建模板
第六节：模板引擎
模板引擎有很多中，Backbone中使用underscore.js为我们提供的模板引擎
语法为：_.template(模板)
在模板中就可以在<%- %> 中添加我们的内容
使用_.templateSettings = { interpolate: /\{\{(.+?)\}\}/g }设置模板语法为{{ }} 代替<%- %> 防止模板引擎冲突
第七节：模板引擎和View
Backbone.View.extend({
template:_.template(模板)
render: function(){
this.template({内容}) } })
第八节：初识模型
初识模型：传入模板引擎的JSON就可以认为是一个模型
第九节：模型
模型：得到或整合要显示到页面中的数据
new model({数据}):直接添加模型数据
defaults:设置默认模型数据
toJSON:把模型转化为JSON形式
set:设置模型数据
get:得到模型数据中的值
第十节：REST
REST：把服务器看成一个程序，不需要知道它怎么实现，只需要了解它的接口
urlRoot: 设置模型访问服务器的url地址
url: 设置url并在后面加上模型的id【如/url/id】
save: 向数据库存入模型
destroy: 删除数据库中的模型
fetch: 得到数据库中的模型
第十一节：Model和View
【见视频】
第十二节：MVC设计模式
MVC是代码文件的归档方式【MVC是一种设计模式】
M：Model模型
V：View 视图
C：Control 控制
如果没有MVC任何应用程序都能实现
MVC没有标准的规范
良好的MVC有助于代码的维护和管理
第十三节：集合 Collection
集合用于管理多个模型
add:向集合中添加模型
remove:删除集合中的模型
第十四节：事件 Event
事件存在于任何对象中，通俗的讲：就是调用函数的另一种方式
on: 监听事件
trigger: 触发事件
第十五节：路由 Router
Router: 可以说就是使用hash调用不同的方法
Router一般和Histroy一起使用
第十六节：SPA
SPA：Single-page Appliction 单页面应用
HTML代码都写在一个文件中，然后通过路由显示不同的页面
第十七节：MV*模式
MV*模式是MVC模式的父集，或者说MV*模式包括MVC模式
M: Model V:View
* 在操作系统中一般表示通配符，在这里表示可以代表任何
