Docker课程概述

Docker让程序快速的部署，高效的运行，神奇的移动

第一节：Docker简介
Docker的作用就是让程序在任何时间，任何地点，任何计算机上高效的运行
第二节：Docker基本操作
docker [选项] [命令名称] [参数]
docker -v 显示当前docker的版本
docker -help 显示docker的命令
直接输入docker显示所有docker的命令
第三节：容器
docker的容器提供了程序运行所需的代码，配置文件，库文件，runtime, 系统工具的文件
docker container ls 查看所有容器
docker container stop 容器名称 停止运行容器
docker container restart 容器名称 重新运行容器
docker container rm 容器名称 删除容器
docker container prune 删除全部停止的容器
第四节：镜像
docker通过镜像运行容器
可以把镜像（image）比喻成运行docker容器的原代码
docker image 显示镜像相关命令
docker images 查看所有镜像
docker rmi 删除镜像
docker history 镜像名称 查看镜像构件历史
第五节：创建镜像
docker可以通过3种方式创建镜像：
docker import 【创建基础镜像（base images)】
docker commit 【创建镜像，和版本控制系统的commit很相似，但此方法在docker中创建image不是很常用】
Dockerfile 【最强大最常用的创建image的方法】
注：一般情况下我们是从已有的image上建立新的image
第六节：Dockerfile
Dockerfile用于创建镜像
FROM: 指定一个已有的镜像，我们创建的镜像将基于此镜像进行修改
RUN:运行镜像中的命令【在新镜像创建中】CMD:运行镜像中的命令【在新镜像创建后】
COPY:把本地文件复制到镜像文件系统中 ADD: 和COPY一样，但能复制tar文件并解压
WORKDIR:改变当前的工作路径
第七节：镜像层级
在docker中，镜像是由镜像层级组成
一个镜像层级就是一个对原有镜像文件改变的集合
在docker中，image可以共享layer
第八节：build和run
build命令格式：docker build [OPTIONS] PATH | URL | -
其中options是 -t 可以指定镜像名称 若果加上":"表示镜像的版本
PATH: 表示指定目录中的Dockerfile文件创建镜像
url: 自定url地址使用dockerfile创建镜像【如 git】
- : 通过标准输入创建镜像 【注：ctr + d 表示结束标准输入】
run命令格式：docker run [OPTIONS] IMAGE[:TAG|@DIGEST] [COMMAND] [ARG...]
可以在run命令后面直接运行容器的命令【如：docker run 镜像名称 ls】
run中的 -t 选项: 表示开启docker的伪终端
run中的 -i 选项: 表示开启标准输入
run中的 -d 选项: 表示后台运行docker
--name: 为运行的容器取名字
第九节：Volume
Volume 是dockerfile中的指令
Volume 可以让容器使用主机的文件
Volume 在Dockerfile中的设置：VOLUME ["挂载路径"]
Volume 在run时设置：-v 主机的绝对路径:挂载路径
Volume的优势
1. 可以让容器使用大的文件
2. 可以让容器与其他容器分享文件
3. 容易设置频繁改变的文件
第十节: 容器网络
docker的网络为的就是docker如何和主机连接，如何和其他的容器连接，以及如何作为服务器
计算机网络的基础是：协议 网络接口 和端口
网桥（Bridge): 就是把所有的网络结合一起看成一个新的网络
每个容器都有一个网络系统
容器与容器之间连接通过docker0网桥分配的私有ip进行连接
容器与主机之间连接通过端口号映射进行连接
EXPOSE: 在dockerfile中曝光容器的端口号
docker run -p 曝光的容器端口:映射的主机端口 镜像名称 通过映射的端口运行镜像
第十一节：发布镜像
在www.docker.com中注册用户以后，可以通过以下步骤发布镜像：
1. docker login 登录用户
2. Username 输入用户名
3. Password 输入密码
4. Docker push 用户/镜像名称 把镜像发表到dockerhub中
5. docker logout 退出登录
docker search 镜像名称：从dockerhub中找到相应的镜像
docker pull 镜像名称：从dockerhub中得到相应的镜像
注：创建发表的镜像最好在dockerfile中添加MAINTAINER指令：用于指定维护者的联系方式
第十二节：容器组合
容器之间可以通过docker0进行连接
我们可以在主机上逐个运行多个容器，也可以通过YML文件一次性运行多个容器
YML文件的优势在于可以让其他开发人员清晰的看清docker中多个容器运行的逻辑
YML文件名称：docker-compose.yml
通过YML文件运行多个程序: docker-compose up
第十三节：docker架构
可以把docker看成由三个部分组成：
docker CLI: 也就是docker接收命令的地方
docker daemon: docker在主机'后台'上运行的服务
docker hub: 存放docker镜像的地方
当我们安装docker是就会默认安装docker CLI和docker daemon
docker CLI 和 docker daemon 以及docker hub是通过http协议连接的
远程连接docker daemon需要配置/etc/init/docker.conf文件【ubuntu】
DOCKER_OPTS='-H tcp://0.0.0.0:2375 -H unix:///var/run/docker.sock'
